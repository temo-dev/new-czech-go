# Feature: Uloha 4 Choice Practice

## Purpose
This feature lets the learner open a choice-and-reasoning task, review the scenario prompt plus available options, record one spoken attempt, and receive transcript plus feedback through the same attempt pipeline already used by the earlier oral tasks.

## User Flow
1. Learner opens the Flutter app.
2. Learner opens a `Uloha 4` exercise from the module list.
3. Learner reviews the scenario prompt and the listed options.
4. Learner records one spoken attempt explaining the chosen option.
5. Backend transcribes, scores, and returns transcript plus feedback.

## Current Status
Implemented today:
- typed backend contract for `Uloha4Detail`
- seeded `Uloha 4` sample exercise detail in the backend store
- admin create flow in the CMS for `Uloha 4`
- CMS edit/delete flow for `Uloha 4`
- CMS local image upload plus preview for `Uloha 4` prompt assets
- `choiceOptions` lines can now carry an optional `image_asset_id` column
- Flutter parsing for `scenario_prompt`, `options`, and `expected_reasoning_axes`
- learner UI rendering for the choice prompt, options, and reasoning hints
- learner UI now renders optional choice images when a `choiceOption` carries an `image_asset_id`

Still simplified:
- scoring still uses the current heuristic choice evaluation
- prompt assets currently live in local CMS/backend storage for authoring preview, not a production object-storage flow

## Risks
- `Uloha 4` still needs optional image assets and richer prompt presentation before it fully matches the intended exam desk
- choice feedback is still generated by the shared V1 scoring path, not a richer choice-specific worker

## Next Step
Now that all four V1 oral task types are testable, the next practical step is to improve cross-task polish: content editing, media assets, or clearer learner-side result/error states.
